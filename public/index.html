Import Guide

导入项目说明

导入前先看，先选对入口，再决定是去 GPT 整理、去项目 repo 的 Codex 会话扫描，还是回总控台确认写入。

导入前先看

本总控台支持两种导入方式：

  • 新项目建档：适合还没正式开始、还没有完整 repo 状态、想先把项目目标和版本边界定清楚的项目。
  • 已有项目建档：适合已有 repo、项目已经做了一部分、想恢复整体项目画像和当前切片位置的项目。

这套导入流程怎么工作

总控台不会替你直接完成所有事情。它负责收集和展示关键项目信息、显示当前缺口、生成给 GPT 或 Codex 的标准提示词，并在你确认后，把项目信息写入项目控制文件。

其中：GPT 负责整理你脑子里模糊、但 repo 里扫不出来的信息；Codex 负责基于项目 repo 扫描、补全结构化项目状态；总控台负责承接、确认和展示结果。

Codex 会在工作前读取项目里的 AGENTS.md，而可复用工作流更适合放进 skills；这是官方推荐的定制方式。

导入路径一：新项目建档

适用场景：你还没有正式推进这个项目，或者 repo 里还没有足够的信息让 Codex 可靠扫描。

你需要做什么：先填写项目名称、项目一句话定义、终版目标、当前版本目标、当前版本不做什么、项目类型、目标用户 / 目标体验、技术偏好 / 技术约束。

如果是游戏项目，再补：游戏分类、核心玩法一句话、画面方向、是否预期有后端 / 联网。

如果你说不清：使用“GPT 辅助整理”。它不会直接写入项目，而是帮你把模糊想法整理成可确认的草案。

完成后会发生什么：确认后的信息会写入项目控制文件，用于建立第一版项目画像和版本边界。这一步不修改业务代码，只建立项目控制材料。

下一步：把系统生成的 Codex 补结构提示词发到项目 repo 的 Codex 会话里，让 Codex 补模块地图、第一工作包、验证口径草案、当前可动作状态。

导入路径二：已有项目建档

适用场景：项目已经做了一部分，但现在看不清项目整体是什么、当前版本目标是什么、当前工作包在哪、当前切片和整体有什么关系、当前卡在哪。

你需要做什么：填写最少恢复信息：项目路径、你当前对项目状态的粗判断、你当前最关心的问题。

接入后会发生什么：总控台会先接入项目，并为你生成一段 Codex 扫描补全提示词。

这段提示词怎么用：把它复制到项目 repo 的 Codex 会话中执行。Codex 会基于 repo 事实补齐项目定义草案、模块地图草案、技术栈草案、版本状态草案、当前切片位置、待确认问题。

注意：Codex 扫描补全不是在总控台里执行，而是在项目 repo 的工作环境里执行。

GPT 辅助整理是什么

适合这些情况：你知道项目想做什么，但说不清；你知道当前版本大概目标，但边界不清楚；你知道游戏大概方向，但玩法 / 风格 / 目标体验还没讲明白；你看完 Codex 扫描结果后，仍有一些关键字段拿不准。

它做什么：帮你把模糊想法整理成可确认的结构化草案；帮你把 provisional 结果收敛成更清晰的描述；帮你形成后续可写入项目控制文件的答案。

它不做什么：不直接修改项目代码，不直接写入项目控制文件，不替你拍板最终方向。

Codex 扫描补全是什么

适合这些情况：repo 里已经有内容；需要把当前项目状态从 repo 中提取出来；需要补齐模块、技术栈、当前工作包、测试状态、变更摘要等结构化信息。

它做什么：扫描 repo 和项目控制文件；读取 AGENTS.md；基于事实补全 .codex-control/ 中的源状态文件；输出待确认问题；不确定的地方标记 needs_confirmation。

它不做什么：不替你拍板产品目标，不擅自扩大范围，不在信息不足时脑补完整项目定义。

三者分工

总控台

  • 入口
  • 表单
  • 状态展示
  • 缺口显示
  • 提示词生成
  • 结果确认
  • 项目控制信息写入

GPT

  • 整理模糊信息
  • 帮你收敛项目定义、版本目标、边界和体验描述

Codex

  • 基于 repo 扫描事实
  • 补全结构化项目状态
  • 更新项目控制文件
  • 把当前切片挂回整体

你现在应该怎么选

  • 选“新项目建档”：这个项目还没开始，repo 里没有足够事实，你需要先把方向和边界讲清楚。
  • 选“已有项目建档”：这个项目已经做到一半，repo 里已经有代码和材料，你想快速恢复当前整体状态。
  • 选“GPT 辅助整理”：你知道大概方向，但说不清；或者看完 provisional 结果后，仍有关键问题拿不准。

导入完成后，你应该看到什么

  • 这项目是什么
  • 终版目标是什么
  • 当前版本目标是什么
  • 当前切片在哪个模块
  • 当前卡在哪
  • 当前最适合的动作是什么
  • 还缺哪些信息
  • 下一条给 Codex 的指令该围绕什么组织

注意事项

  • 总控台不会自动替你做产品拍板
  • GPT 不会自动成为项目真相来源
  • Codex 不应在信息不足时替你定义战略目标
  • 日常自动聚合仍然只读项目 repo
  • 只有你显式确认的动作，才会写入项目控制文件
